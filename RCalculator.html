Using R as a calculator

1 Basic calculations

As R is a programming language, it responds to series of commands. Are written into the Console (underneath your Script pane). Primarily it can be used a calculator. For example:

Code 
2 + 2
[1] 4

Here, the equation 2 + 2 is inputted into the console. To execute (or run) this, you can click anywhere along the line of code and click on the run button:

Alternatively, you can hit the following combination of keys on your keyboard when your cursor is place anywhere along that line:

  • Windows: Ctrl + Enter
  • Mac: Cmd + Enter (or Ctrl + Enter)

Note that the result of the calculation appears in the console.

An advantage of using a script is that you are able input multiple lines of code and run them together. For example:

Code 
2 * 2
[1] 4
Code 
sqrt(16)
[1] 4
Code 
abs(-4)
[1] 4
Code 
log2(16)
[1] 4

Running these together returns a list of outputs.

Tip

sqrt(...), abs(...), and log2(...) are base functions featured in R. These and others will be discussed in more detail later in this guide.

  1. Use R to calculate the following equations:

\[ 2 \times5+6 \]

\[ 30 \times 30 \times 60 \]

\[ \sqrt{\frac{40}{12} \times 50} \]

2 Brackets

Brackets play an import role in R. Fundamentally, they are applied to isolate a calculation or sequence. Consider the following example:

Code 
2 + 2 * 4
[1] 10

Versus:

Code 
(2 + 2) * 4
[1] 16

When a function such as sqrt is entered, it will contain brackets within which you can specify the values that you want to test. For example:

Code 
sqrt(16) - sqrt(4)
[1] 2

3 Creating Objects: R’s Memory feature.

Objects are a useful way of containing data including variables and, later on, user-defined functions. They can be assigned unique names but these must not include spaces or, where possible, override existing functions in R.

Objects can created by using a format similar to result = 2 + 2. You can also store text strings in an object for example text_result = "Jane".

An alternative to using the = symbol is <- that is particularly use when you later start to incorporate packages into your coding.

3.1 Creating a vector

A vector is a string of values assigned to an object. It is a fundamental feature when it comes to statistical processes. To do this we need to introduce concatenation to denote a series of things, for instance a sequence of numbers. This is enabled using the concatenation function c(...). Here is an example of how it is applied to a series of numbers:

Code 
my_numerical_vector <- c(1,2,3,4,5,6,7,8,9)

An example of when applied to a series of characters (text values); double or single quotations convert denote characters:

Code 
my_character_vector <- c("Alice", "Bertha", "Collin", "David", "Ethan", "Francis", "Georgina", "Helen", "Iain")

The examples given above contain a couple of other note worthy features.

  • A designated name for your object. This is the my_vector part of the coding line.
  • An operator that looks like <- that is interchangeable with = sign. Here, we are telling R that our object equates to our specified numerical (or string) vector.
Tip

Each entry along the figure is separated with a comma ,

Whenever you create an object, it appears in your the Global Environment located immediately to the left of your R Script window.

When you combine numeric and string values into the same vector, its classification becomes ‘character’ rather than numeric.

When the <- operator points the other way (->), this denotes that code is being pushed towards an object; typically, this operator is preceded by a pipe %>% operator that consists of a sequence of actions to be discussed later!.

To create a matrix you’ll first need a two vectors. Let’s take the monthly average temperatures recorded in Waddington during 2020, 2022, and 2023 as an example @. The first vector, months, will contain abbreviated calendar months; the second vector will contain relative temperature (high)values in Celsius (ºC), see Table 1. The code for these is as follows:

Code 
months <- c("Jan", "Feb", "Mar", "Apr", "May", "Jun", "Jul", "Aug", "Sep", "Oct", "Nov", "Dec")

Year_2020 <- c(9.2, 9.7, 10.6, 15.7, 18.4, 19.8, 20.2, 22.0, 18.9, 13.4, 11.4, 6.9)
Year_2021 <- c(5.1, 7.9, 11, 11.2, 14.5, 19.9, 22.5, 20.3, 20.7, 15.3, 10.6, 8.3)
Year_2022 <- c(7.6, 10.0, 12.1, 13.9, 17.8, 20.7, 24.5, 24.5, 19.0, 16.7, 11.3, 6.3)

It is work noting how they are presented for example [1,] or [,1]. As stated earlier, the two-dimensions present here are rows and columns, therefore, think of the brackets as showing [rows, columns], the number numbers act as coordinates for your data. Using the dim(...) function, it is possible to preview the dimensions of a respective matrix.

From this result, we learn that the matrix has 12 rows and 3 columns.

3.2.2 Renaming columns/rows

The names in the Temperature_matrix aren’t well formatted (i.e. Year_2020) - we want to remove the underscore between Year and 2020, and do the same for the other columns. Here, we can use the colnames(...) function to override these names. The process involves the manipulation of the matrix’s dimensions and is as follows:

Code 
colnames(Temperature_matrix)[1] <- "Year 2020"
colnames(Temperature_matrix)[2] <- "Year 2021"
colnames(Temperature_matrix)[3] <- "Year 2022"

4 Introduction to summary statistics

With vector and matrix data stored in R’s Global Environment, it is possible to examine it with summary statistics. In R, this is relatively straightfoward so long as you have a good grounding in the statistical arm of mathematics. Perhaps the simplist function is the sum(...) function that work in exactly the way as it does in Microsoft Excel.

If, for some reason, you wanted to calculate the total temperature for the vector Year_2020, you’ll do the following:

Code 
sum(Year_2020)
[1] 176.2

Using the mean(...) function, it is possible to calculate the average temperature for that year, that may be a far more useful exercise…

Code 
mean(Year_2021)
[1] 13.94167

However, what if we wanted to calculate the average temperature for the month of August across the years 2020, 2021, and 2022? We are able to conduct the same function on the matrix, taking care to specify the correct dimensions:

Code 
# Obtaining the correct 'slice' for our data:
Temperature_matrix[5,]
Year 2020 Year 2021 Year 2022 
     18.4      14.5      17.8
Code 
# Calculating the average by wrapping the slice with the mean(...) function
mean(Temperature_matrix[5,])
[1] 16.9

At this point, it’s taking note of the summary(...) function in R. This function flexibly provides a collection of useful summary statistics for a vector, matrix array, or a data frame.

When applied to Year_2022 vector, the following information is shown.

Code 
summary(Year_2022)

   6.30   10.97   15.30   15.37   19.43   24.50

Here, we have the minimum value, the first and third quartile, median (middle of the data), and the mean average.
